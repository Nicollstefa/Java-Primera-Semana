## Actividad 1: Identificación de Objetos (Ejercicio 01)

### Objetivo
Identificar objetos del mundo real dentro del dominio de una clínica dental.

---

## Objeto 1: Paciente Dental
- **¿Qué es?**  
  Persona que recibe servicios odontológicos en la clínica.  
- **Características (atributos):**  
  nombre, cédula, edad, teléfono, historial dental, última cita  
- **Comportamientos (métodos):**  
  agendarCita(), consultarHistorial(), actualizarDatos(), generarFactura()

---

## Objeto 2: Odontólogo
- **¿Qué es?**  
  Profesional encargado de realizar los tratamientos dentales a los pacientes.  
- **Características:**  
  nombre, especialidad, número de licencia, horario, consultorioAsignado  
- **Comportamientos:**  
  atenderPaciente(), registrarTratamiento(), actualizarHorario()

---

## Objeto 3: Cita Odontológica
- **¿Qué es?**  
  Reservación programada entre un paciente y un odontólogo para un tratamiento.  
- **Características:**  
  fecha, hora, paciente, odontólogo, motivo  
- **Comportamientos:**  
  confirmarCita(), cancelarCita(), modificarHora()

---

## Objeto 4: Consultorio
- **¿Qué es?**  
  Espacio físico donde se atienden los pacientes y se realizan los tratamientos.  
- **Características:**  
  número, tipoDeEquipo, estado, odontólogoAsignado  
- **Comportamientos:**  
  asignarOdontólogo(), verificarDisponibilidad(), limpiarConsultorio()

---

## Objeto 5: Factura
- **¿Qué es?**  
  Documento que detalla los servicios prestados y el costo correspondiente.  
- **Características:**  
  númeroFactura, paciente, tratamientos, valorTotal, fechaEmisión  
- **Comportamientos:**  
  calcularTotal(), generarPDF(), registrarPago()

# Actividad 2: Comparación de Paradigmas (Ejercicio 02)

### Objetivo
Comprender las diferencias entre **programación estructurada** y **programación orientada a objetos (POO)** aplicadas al contexto de una clínica dental.

---

## Programación Estructurada

La **programación estructurada** se basa en el uso de variables y funciones independientes.  
No agrupa datos ni comportamientos dentro de una misma entidad.  
Cada dato del programa se maneja de forma individual, lo que puede dificultar la organización cuando aumenta la complejidad.

---

### Ejemplo Conceptual

En este ejemplo, se muestra cómo se manejaría la información de un paciente dental en un programa estructurado:

```java
// Manejo estructurado de un paciente dental

// Variables sueltas
String nombre = "María Rodríguez";
String id = "1015438920";
String telefono = "3102345678";

// Función para mostrar los datos del paciente
void mostrarPaciente() {
    System.out.println("Paciente: " + nombre);
    System.out.println("Cédula: " + id);
    System.out.println("Teléfono: " + telefono);
}
